# インデックスバッファ

インデックスバッファとは、頂点バッファ内の**どの頂点を使って図形（ポリゴン）を組み立てるか**を、頂点番号（インデックス）のリストとして指示するGPUメモリ上のバッファです。

---

## 1. なぜインデックスバッファが必要なのか？（課題と解決）

前回の「頂点バッファ」では、3つの頂点を使って1つの三角形を描画しました。
では、**四角形（矩形）**を描画したい場合はどうすればよいでしょうか？

GPUは基本的に「三角形」の組み合わせで図形を描画します。そのため、四角形は**2つの三角形**に分割して描画する必要があります。

```plaintext
   V1          V2
    ┌──────────┐
    │ ＼      2 │
    │   ＼      │  三角形①: [V0, V1, V2]
    │      ＼   │  三角形②: [V2, V3, V0]
    │ 1      ＼ │
    └──────────┘
   V0          V3
```

### 頂点バッファだけで四角形を描く場合（非効率）

インデックスバッファを使わない場合、GPUに2つの三角形分の頂点データをそのまま渡す必要があります。
つまり、合計 **6つの頂点データ** を用意しなければなりません。

- **頂点バッファの内容**: `[V0, V1, V2, V2, V3, V0]`
- **無駄**: `V0` と `V2` が **2回ずつ重複** しています。

これが複雑な3Dモデル（何万ものポリゴンを持つキャラクターや地形など）になると、1つの頂点が5〜6個の三角形で共有されることも珍しくありません。頂点データには「座標」だけでなく「色」「法線」「UV座標（画像テクスチャの位置）」など多くの情報が含まれ、1頂点あたり数十〜数百バイトになります。
これらをすべて重複してメモリ（VRAM）に格納し、GPUのバス（通信経路）に流すのは**極めて非効率**です。

## 2. インデックスバッファによるスマートな解決策

インデックスバッファを使うと、**「頂点データそのもの」と「それらを結ぶ順番（設計図）」を完全に分離**できます。

### ① 頂点バッファ (Vertex Buffer)

重複のない、ユニークな頂点データだけを格納します（四角形なら4つだけ）。
*   `[V0, V1, V2, V3]` (要素数: 4)

### ② インデックスバッファ (Index Buffer)

頂点バッファの「何番目の頂点を使うか」を表す整数（インデックス）の配列です。
*   `[0, 1, 2,  2, 3, 0]` (要素数: 6)
    *   前半 `[0, 1, 2]` で 1つ目の三角形（V0 - V1 - V2）
    *   後半 `[2, 3, 0]` で 2つ目の三角形（V2 - V3 - V0）

> [!TIP]
> **メモリ削減効果の例**
> *   1頂点 = 32バイト (座標 + 色) とします。
> *   **頂点バッファのみ**: 6頂点 × 32バイト = **192バイト**
> *   **インデックスバッファ併用**: (4頂点 × 32バイト) + (6インデックス × 2バイト※u16) = 128 + 12 = **140バイト**
> 
> モデルが複雑になればなるほど、この差は劇的に広がります（通常、数倍〜十数倍のVRAM節約になります）。

## 3. GPUキャッシュの恩恵 (Vertex Cache)

インデックスバッファを使用するもう1つの強力なメリットは、GPUの**「ポスト・トランスフォーム・頂点キャッシュ (Vertex Cache)」**を利用できる点です。

GPUは頂点シェーダーで「頂点座標の計算」を行います。

*   インデックスバッファを使っている場合、GPUは「いま計算した頂点インデックス」をキャッシュに一時保存します。
*   次に同じインデックスの頂点を処理する際、**頂点シェーダーの再計算をスキップし、キャッシュから結果を再利用**します。
*   これにより、描画パフォーマンスが飛躍的に向上します。

## 4. `wgpu` における実装の変更点

wgpuでインデックスバッファを導入する際、実装は以下の3点が変化します。

### ① インデックス用のバッファ作成

頂点バッファと同様に `device.create_buffer_init` を使用して作成しますが、用途（`usage`）に `wgpu::BufferUsages::INDEX` を指定します。

```rust
let index_buffer = device.create_buffer_init(
    &wgpu::util::BufferInitDescriptor {
        label: Some("Index Buffer"),
        contents: bytemuck::cast_slice(INDICES),
        usage: wgpu::BufferUsages::INDEX, // インデックスバッファ用
    }
);
```

### ② インデックスのデータ型 (Index Format)

GPUがインデックスバッファを読み込む際、1つのインデックスが何ビットか（`wgpu::IndexFormat`）を指定する必要があります。

*   `wgpu::IndexFormat::Uint16`: 16bit整数 (`u16`)。最大 65,535 頂点まで扱える。メモリ効率が良い。
*   `wgpu::IndexFormat::Uint32`: 32bit整数 (`u32`)。それ以上の膨大な頂点数を扱う場合に必要。

### ③ 描画コマンドの変更

コマンドエンコーダー（RenderPass）での描画指定が、通常の `draw` から `draw_indexed` に変わります。

```rust
// 1. 頂点バッファをセット (スロット0)
render_pass.set_vertex_buffer(0, self.vertex_buffer.slice(..));

// 2. インデックスバッファをセット (データ型を指定)
render_pass.set_index_buffer(self.index_buffer.slice(..), wgpu::IndexFormat::Uint16);

// 3. インデックスを使って描画命令を発行
// 引数: (インデックスの描画範囲, 頂点バッファの開始オフセット, インスタンスの範囲)
render_pass.draw_indexed(0..self.num_indices, 0, 0..1);
```
