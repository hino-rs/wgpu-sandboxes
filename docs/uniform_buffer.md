# ユニフォームバッファ

Uniformバッファは、GPUで描画を行う際に「すべての頂点やピクセルで共通して参照したいデータ」を渡すための仕組み。これを利用することで、オブジェクトを動かしたり、時間経過で色を変化させるといった、**動的な表現**が可能になる。

## Uniformバッファとは？

- 頂点バッファ: 
    - 頂点ごとに異なるデータが入った配列。
    - シェーダー内では「頂点シェーダの入力引数`location(n)`」として、頂点ごとに自動で次の要素が読み進められる。
- Uniform バッファ:
  - 全ての頂点・全てのピクセルで「全く同じ値」として参照される単一のデータ空間。
  - プログラミングで言うグローバル定数のようなもの。
  - 主な用途: 変換行列、カメラのビュー・プロジェクション行列、経過時間、ライトの方向や色、画面解像度など。

## 動的なデータ送信のメカニズム

1. CPU側で構造体のデータを更新
2. `queue.write_buffer`でGPUへ転送
3. `set_bind_group`でバッファをバインド
4. GPUが新しい値を使って描画
5. 1へ

```rust
queue.write_buffer(&uniform_buffer, 0, bytemuck::bytes_of(&my_data));
```

- 内部で適切にステージングが行われ、GPUのメモリに対して非常に効率的かつ安全にデータを上書きする。
- この操作を行うバッファは、作成時に`wgpu::BufferUsages::COPY_DST`(コピー先になれる権限)を付与しておく必要がある。
  
## wgpuにおける実装の全体像

### 1. CPU側のデータ構造の定義

GPUに送るデータ構造をRust側で定義する。
頂点バッファと同じく、メモリアライメントには注意する。

```rust
#[repr(C)]
#[derive(Debug, Copy, Clone, bytemuck::Pod, bytemuck::Zeroable)]
struct MyUniform {
    translation: [f32; 2], // X, Y の移動量
    _padding: [f32; 2],    // 16バイトアライメントのためのパディング
}
```

### 2. WGSLシェーダー側での受け取り

シェーダー側では、`var<uniform>`を使ってグローバル変数として宣言する。

```wgsl
struct MyUniform {
    translation: vec2<f32>,
}

@group(0) @binding(0) // Bind Groupの何番に紐づくか
var<uniform> u_variables: MyUniform;
```

### 3. バッファの作成

データを格納するメモリ領域をGPU上に確保する。

用途（Usage）には、`UNIFORM`（Uniformとして使う）と `COPY_DST`（CPUから書き込めるようにする）を指定する。

### 4. バインドグループの作成

WGPU（およびWebGPU/Vulkan）では、バッファを直接シェーダーにバインドするのではなく、 **「Bind Group（バインドグループ）」** という仲介役を介してバインドします。

- Bind Group Layout (設計図): 「グループ0のバインディング0番には、Uniformバッファが来ますよ」というルール（型定義）をパイプラインにあらかじめ教えておきます。
- Bind Group (実体): 設計図に従って、「具体的にどのバッファをバインドするか」を決定した実体です。

## メモリアライメント規則

GPU(WGSL)は、メモリにアクセスする効率を高めるために、データの配列(バイト境界)に厳しいルールを持つ。

基本ルール:

1. `vec4`(16バイト)や`mat4x4`(64バイト)は、16バイトの境界に配置されなければならない。
2. Uniformバッファの全体サイズは、16バイトの倍数でなければならない。
