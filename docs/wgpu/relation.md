# ざっくり

1. Windowsでゲームを遊ぶとき、ウィンドウが展開され、そこに映像が描画される。

2. その時、描画を担うのは**Direct3D**。
Direct3Dは、**グラフィックスAPI**の1つで、Windowsを前提に作られているもの。
グラフィックスAPIは、GPUを動かして映像を描画するためのAPI。

3. **WebGPU**は。グラフィックスAPIへのアクセスを抽象化したAPI。つまり、Direct3Dのような「○○向けグラフィックスAPI」ではない。GPUと直接やりとりするような仕様でもない。ただし、「ブラウザで使うこと」を前提に設計されている。

4. Rustのクレートである`wgpu`は、WebGPUの仕様を、ブラウザ外(Windowsなど、ネイティブアプリ)でも使えるように実装したもの。つまり、「ブラウザ,Windows, Mac, Linux」と、どこでも動かすことができるように作られている。

5. **WGSL**は、WebGPU仕様の一部として策定した**シェーダー言語**。シェーダー言語は、GPUへの指示書用言語的なもの。`main.rs`がCPUに向けた指示書なら、`shader.wgsl`は**GPUに向けた指示書**。
